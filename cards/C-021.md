## Scope

The Review gate (`adversarial-review.md`) routes by SECURITY class (C-014) but is otherwise
LANGUAGE-blind ŌĆö every card gets the generic `code-reviewer`. The host registry ships
`typescript-reviewer` (type-safety, async correctness, Node/web idioms) and `python-reviewer`
(PEP8, type hints, Pythonic idioms) ŌĆö strictly more rigorous on their language than the generic
reviewer. The detection primitive already exists (`cmd_assess` fingerprints node/python/go/rust
via package.json/pyproject/go.mod/Cargo.toml at flow.sh:1051-1055), and a card's language is
directly readable from its `## Allowed files` extensions.

Fix ŌĆö add a **language-specialist lens** to `adversarial-review.md`, mirroring the security-lens:
- Detect the card's primary language from its changed-files extensions: `.ts/.tsx/.js/.jsx` ŌåÆ
  `typescript-reviewer`; `.py` ŌåÆ `python-reviewer`; otherwise generic `code-reviewer`.
- The specialist is **layered with** `code-reviewer` (not a replacement), and **composes with**
  the security lens (a security-class TS card ŌåÆ security-reviewer + typescript-reviewer + code-reviewer).
- **Gate parity**: informs triage; NEVER auto-passes/auto-fails (same rule as the security lens).
- **Detect-first degrade**: specialist AGENT present ŌåÆ use it layered; absent ŌåÆ generic
  `code-reviewer` (the gate is identical on every rung ŌĆö a missing specialist only changes who
  reviews, never whether the card passes).
- Wire `typescript-reviewer` + `python-reviewer` into `agent-detection.md` and the
  `agent-stage-mapping.md` Review row so they are first-class (and the C-018 tripwire, which derives
  its required set from `agent-detection.md`, GUARDS them ŌĆö they MUST appear in the mapping or the
  tripwire goes red). VERIFY the tripwire stays green after wiring.

Docs/test layer only ŌĆö the routing is an instruction to the Review semantic layer (Claude), like
the security lens; no runner code needed.
