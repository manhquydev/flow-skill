# Retro

After all cards are done, append ONE line per project run:
*Which gate did I skip or rush, and what did it cost?*

**Optional aid — the `retro` SKILL (opt-in).** Before writing the line, you may offer the `retro`
skill (git-history-driven: commit/velocity/code-health numbers) so the retro is backed by real
figures, not memory. It INFORMS the operator's line — **the operator still writes the line; the
skill never authors it** (the teach-mode rule holds). Distinct from the `journal-writer` agent
(narrative) — see `claudekit-skills.md`. After it runs at this gate, record the lazy durable
metric via `flow.sh harness intervention add`.
