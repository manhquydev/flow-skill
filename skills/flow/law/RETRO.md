# Retro

After all cards are done, append ONE line per project run:
*Which gate did I skip or rush, and what did it cost?*

**Run the native numeric-retro ritual first (guaranteed baseline — `native-rituals.md`
§4, no external skill required).** Before writing the line, walk the git-log recipes in
the ritual (commit count, churn, per-stage dwell via `flow.sh usage`, cadence) so the
line is backed by real figures, not memory.

**If the `retro` skill is installed**, offer it as a richer alternative (opt-in) — same
git-history-driven idea, plus code-health indicators. It INFORMS the operator's line —
**the operator still writes the line; no ritual or skill ever authors it** (the
teach-mode rule holds). Distinct from the `journal-writer` agent (narrative) — see
`claudekit-skills.md`. After either runs at this gate, record the lazy durable metric via
`flow.sh harness intervention add`.
