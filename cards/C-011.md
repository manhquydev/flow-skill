# C-011 — global per-stage dwell (compact line carries stage_from)

status: done

## Scope

`usage --global` reports per-stage dwell as empty because the COMPACT device-global log line
(`flow.sh:1423`) omits `stage_from` — it carries only `stage_to`. The dwell reconstruction
(`flow_harness.py:567-613`, shipped v0.11 FR3) needs BOTH `stage_from` epoch (enter) and
`stage_to` epoch (exit) from `next` transitions; starved of `stage_from`, the global view
returns no per-stage dwell rows. Fix (**HYBRID**, operator-chosen):
- **(A) new rows exact.** Add `stage_from` to the compact line so post-fix events reconstruct via
  the IDENTICAL project-local code path (`flow_harness.py:567-613`) — zero risk of a divergent
  "second algorithm". This is the source of truth.
- **(B) legacy rows best-effort.** For lines lacking `stage_from`, INFER it from the preceding
  in-cycle row's `stage_to`, **partitioned by `(project, cycle_id)` and ordered by `epoch_s`** (never
  by physical line order; never across cycles/projects). Mark these dwell rows as approximate.
  Result: legacy log is no longer blind, while new rows stay exact.

Inference must NOT silently disagree with the exact path: where a real `stage_from` exists it
always wins; inference fires only when the field is absent. Re-entries / multi-day idle gaps in the
legacy fallback are acknowledged-approximate, not authoritative.
