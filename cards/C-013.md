## Scope

The auto-run repair ladder (`auto-run.md:25-26`) escalates a red card to "a FRESH same-ladder
(Claude) subagent" — a *generic* fresh subagent, not the purpose-built `debugger` agent that
exists in the registry (root-cause investigation, log/test analysis, diagnostic reports). A
generic drafter re-attempting the same implementation is weaker than a diagnostic agent at strike
2. Fix: wire the `debugger` agent into the repair loop so strike-1 (or strike-2) repair runs
`Task(subagent_type="debugger")` with the scoped failing-card brief BEFORE the cross-engine
(Codex) escalation. Keep the existing escalation order intact: debugger (Claude diagnostic) ->
Codex fresh-engine (if USABLE) -> Antigravity -> operator. Markdown/orchestration layer only; no
runner edit.

**Portability law (must hold):** this is a detect-first ladder entry, NOT a hard dependency. If
`debugger` is absent in the host, degrade — strike repair runs inline root-cause then the existing
fresh-subagent redraw. A missing `debugger` changes WHO diagnoses, never whether the card's
`## Verify` + `flow.sh check` must pass for real. State this degrade rung explicitly in the doc.

Context isolation (orchestration-protocol): debugger gets task + failing card file + test output
+ acceptance — NO session history.
