# C-017 — telemetry honesty polish (legacy-dwell label + dead-var)

status: done
deps: none
group: v0.12.1-polish
implements: P-LOW1, P-LOW2

## Scope

Two cosmetic honesty fixes in `flow_harness.py` `cmd_usage`, both surfaced by the v0.12 integration review (LOW, non-breaking):

- **LOW-2 — label approximate legacy dwell.** The per-stage dwell section prints inferred (legacy, pre-v0.12 rows lacking `stage_from`) numbers with no in-band marker. A user could mistake an inflated multi-day-gap legacy average for an exact metric. Fix: when `--global` dwell used the inference fallback for any stage (a row had no real `stage_from`), append a one-line marker to the dwell section header, e.g. `per-stage dwell (wall-clock; ~approx for pre-v0.12 global rows):`. New rows with real `stage_from` stay unmarked/exact. The release theme is "honest telemetry" — an unlabeled approximation is off-theme.
- **LOW-1/#2 — dead var `display_count`.** `--builds-only` correctly filters the cycle-time numbers but its intended "(N build cycles)" count label was never wired, leaving `display_count` assigned-but-never-read. Fix: either WIRE the label (print the filtered build-cycle count on the cycle-time line under `--builds-only`) or REMOVE the dead variable. Prefer wiring the label (completes the advertised flag); removal is acceptable if wiring is awkward.
