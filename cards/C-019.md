# C-019 — engine hygiene: tempdir EXIT trap in advisory probes

status: done
deps: none
group: v0.12.1-polish
implements: P-1

## Scope

**#1 — tempdir leak on signal-kill.** The advisory probe functions in `flow.sh` (`cmd_tokens`,
`cmd_coherence`, `cmd_consistency`) create `mktemp -d` working dirs and `rm -rf` them on every
normal return path — but a SIGINT/SIGTERM mid-run leaves the dir behind (the process-level EXIT
trap only logs; it does not clean caller-local `$td`). LOW impact (one small /tmp dir per kill),
systemic across the 3 funcs. Fix: register cleanup so the tempdir is removed even on signal —
e.g. a `trap 'rm -rf "$td" "$td2"' RETURN` inside each func, or route all temp dirs through a
shared list cleaned by the existing EXIT trap. Preserve current behavior (normal-path cleanup
still works); add nothing to the hot-path commands (next/card/check create no tempdirs).

Note: `cmd_design` is NOT affected (it has no `mktemp`). Scope is exactly the 3 funcs above
(4 tempdir instances incl. `cmd_tokens`'s `$td` + `$td2`).
